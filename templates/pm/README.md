# PM framework templates

These files are **canonical reference templates** for Code-Left’s PM workflow: use them to start from scratch, **or**—when your product org already has its own brief, PRD, and technical-spec shapes—as a **gap check** and refinement guide. Point agent skills at your live templates and compare against these when helpful.

Copy into your product repo or handbook when you need a baseline. Paths in the [`skills/pm/`](../../skills/pm/) skills assume your team either uses similar paths or tells the agent where files live.

## Start here

| Template | Use when |
| --- | --- |
| [bet-brief-template.md](./bet-brief-template.md) | You do not already have a brief / PRD shape for a bet |
| [technical-spec-template.md](./technical-spec-template.md) | You need an implementation-ready spec and do not already have an org template |
| [where-we-want-to-get-to.md](./where-we-want-to-get-to.md), [who-we-are-now.md](./who-we-are-now.md), [how-we-work.md](./how-we-work.md) | You need lightweight context pages; rough links or notes are enough to start |
