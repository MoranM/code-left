# build-technical-spec

Turns a **bet brief** (or org equivalent) into a **standalone technical specification**: user stories, functional requirements, edge cases, success criteria, and an explicit product contract so implementers do not need to re-read the brief for critical behavior.

**Team templates:** Use your org’s technical spec outline when you have one; [`technical-spec-template.md`](../../../templates/pm/technical-spec-template.md) is the Code-Left **reference / gap-check** shape.

## When to use

- After a bet brief exists (preferred) or with explicit intent for spec-only mode.
- When the brief changed or stakeholders added comments—update the spec in place.
