# build-product-brief

Builds or improves a **bet brief** (or your org’s PRD / product brief): the product-intent artifact for a feature-sized bet. Uses three context layers (where we want to get to, who we are now, how we work) so agents do not invent strategy or user reality.

**Team templates:** Prefer your existing brief or PRD shape when you have one; the Code-Left [`bet-brief-template.md`](../../../templates/pm/bet-brief-template.md) is a reference for gaps and refinement.

## When to use

- Creating or refining a bet before engineering spec work.
- Aligning PM and stakeholders on scope, promise, and success signals.

## Templates and docs

- Reference (gap check): [`templates/pm/bet-brief-template.md`](../../../templates/pm/bet-brief-template.md)
- Framework: [`docs/pm-framework/README.md`](../../../docs/pm-framework/README.md)

## Default artifact name

`bet-brief.md` in the feature folder is a common default. Teams may use another name or tool; tell the agent your path or convention.
