# map-product-context

Helps PMs and product leaders decide **which context to add first** when adopting the Code-Left PM framework (bets plus three context layers).

## When to use

- Starting Code-Left-style delivery with agents.
- Teams have scattered docs, analytics, and tribal knowledge and need a minimal viable context set.
- Before writing many bet briefs.

## Output

Typically `context-map.md` next to your context docs, or inline in chat. See [SKILL.md](./SKILL.md).
